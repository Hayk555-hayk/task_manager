Git это система сохраняющая изменения в наших файлах Позволяет просмотр всех версий файла Позволяет многим людям кооперировать
Для изменения пользователя git нужны следующие команды git config --global user.name someName git config --global user.email email@email Для получения данных нужно прописать git config user.name git config user.email
Простые команды git cd ../smt/smt2 изменения директории ls или dir лист файлов в директории mkdir smth создания новой папки touch index.html создания файла rm index.html Удаления файла rmdir smth удаление директории
Репозитория указывается .git файлом в проекте commit анологичен быстрому сохранению в играх, благодаря этому мы можем возвращатся обратно если что то сломали git init Для создания репазитория git status покажет какие файлы были изменены git add filename или git add . добавит измененные файлы в стейджинг git rm --chached filename Для удаления файла из стейджинга который был добавлен не правильно git commit -m "description of changes" git log покажет все commit которые были сделаны git log --oneline покажет все commit которые были сделаны одной строкой git checkouot some_id_from_git_log мы увидим код во время этих изменений git revert some_id_from_git_log удалит выбранный commit (появится скрин где мы пропишем wq и нажмем enter) git reset some_id_from_git_log удалит все commit после выбранного, код останится на всякий случай и мы сможем его вновь использовать, но если реально нужно удалить то git reset some_id_from_git_log --hard
Ветки создают изолированный enviroment для тестинга кода git branch branchName создание ветки git branch -a показывает все ветки git checkout branchName смена ветки Для удаления ветки переходим на master ветку и вводим git branch -D branchName git checkouot -b branchName создаст и перенаправит на новую ветку
Для обьединения веток переходим в нужную ветку допустим master и прописываем git merge branchName Если возникают конфликты, решаем их и после git add . , git commit, wg enter
Работа с online репазиторием git clone path_of_repository после add и commit добавляется push git push origin branchName Можно обнавлять свою ветку воизбежания конфликтов git pull origin branchName git remote -v покажет origin git remote add origin path_of_repository если нет origin но как правило он автоматически сгенерируется
